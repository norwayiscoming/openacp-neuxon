3 Approaches: AI Progress Graph Plugin

Choose the visualization approach that fits best

A

🗺️ Journey Map — "Con đường của AI"

Metaphor: A road/path from start to finish

Graph flows left → right like a journey. Big milestone nodes on the main road. Side branches for detours (debugging, retrying). New nodes grow organically as AI discovers them.

Pros

  • Very intuitive for non-tech — everyone understands a journey
  • Natural sense of progress (left = past, right = future)
  • Easy to show "how far along"
  • Detours/retries feel natural as side roads

Cons

  • Can get wide/long for complex tasks
  • Hard to show parallel work
  • Less "techy" — may feel too simple for PMs

B

🧠 Mind Map — "Bản đồ tư duy của AI"

Metaphor: A brain thinking out loud

Central node = the task. Branches radiate outward for each phase. Sub-branches for discoveries, decisions, problems. Nodes pulse when active. Feels like watching an AI think.

Pros

  • Great for showing "why" — reasoning is visible
  • Handles complexity well — branches can go deep
  • Feels magical — watching a mind work
  • Good for both progress AND reasoning

Cons

  • Can get messy/cluttered
  • Harder to see linear progress ("how far?")
  • Non-tech users may find it overwhelming

C

🎮 Quest Board — "Bảng nhiệm vụ RPG"

Metaphor: A game quest/mission board

Task = a quest with stages. Each stage is a "level" node. AI moves through them like a character in a game. Side quests for unexpected work. Progress bar + graph hybrid.

Pros

  • Gamification — fun and engaging for everyone
  • Clear progress (stage 3/5)
  • Side quests = perfect metaphor for detours
  • Works great for both PM and grandma
  • "Quest completed!" feels satisfying

Cons

  • May feel "unserious" for enterprise users
  • Harder to show deep reasoning
  • Game metaphor may not fit all contexts

💡 Recommendation: A + C Hybrid

Use the Journey Map (A) as the core graph layout — nodes flowing left to right showing the path. But borrow the Quest Board (C) gamification elements — progress percentage, stage labels, "side quest" naming for detours, satisfying completion animations.

This gives you: intuitive flow (journey) + clear progress (quest) + living graph (nodes grow/branch) + fun for non-tech (game elements).
